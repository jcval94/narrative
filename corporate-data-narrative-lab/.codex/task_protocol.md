# Task Protocol

## Crear Caso Nuevo

1. Leer `DATA_SCIENCE_CURRICULUM.md`.
2. Usar `00_curriculum_mapper`.
3. Usar `01_case_thesis_builder`.
4. Usar `02_corporate_conflict_builder`.
5. Usar `03_data_signal_designer`.
6. Usar `04_synthetic_evidence_generator`.
7. Usar `05_simple_visual_builder`.
8. Diseñar el `Narrative Learning Experience Standard`: `opening_hook`, rol del alumno, misterio, historia oficial, historia oculta, pista visual, pregunta activa, traducción de jerga, giro, decisión débil, decisión robusta, cierre tragicómico y regla transferible.
9. Usar `06_narrative_case_writer`.
10. Usar `07_tragicomic_editor`.
11. Usar `08_learning_designer`.
12. Usar `09_business_decision_simulator`.
13. Usar `10_pilot_designer`.
14. Usar `11_ethics_and_risk_reviewer`.
15. Usar `12_senior_data_reviewer`.
16. Usar `13_html_story_renderer`.
17. Usar `14_quality_gatekeeper`.

## Mejorar Caso Existente

1. Validar estructura con `tools/validate_case_structure.py`.
2. Identificar secciones débiles con la rúbrica.
3. Corregir primero tesis, conflicto y evidencia.
4. Corregir intriga inicial, rol del alumno, pregunta activa, traducción de jerga y regla transferible.
5. Recién después ajustar humor o estilo.
6. Renderizar HTML y revisar que no parezca dashboard.

## Revisión

La revisión debe responder:

- ¿La tesis es clara?
- ¿La técnica mínima suficiente está justificada?
- ¿La gráfica prueba la historia?
- ¿La gráfica funciona como escena del crimen con pista y lectura guiada?
- ¿El alumno investiga activamente?
- ¿La mala decisión es defendible?
- ¿El piloto puede fallar?
- ¿Hay riesgos éticos tratados con seriedad?
- ¿El HTML mantiene foco narrativo?
- ¿El cierre deja regla transferible?
