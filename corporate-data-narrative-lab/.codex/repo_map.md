# Repo Map

## Raíz

- `README.md`: entrada operativa del repositorio.
- `PROJECT_BRIEF.md`: definición del producto educativo.
- `DATA_SCIENCE_CURRICULUM.md`: niveles 0-24 y variación de técnicas.
- `NARRATIVE_PRINCIPLES.md`: estructura narrativa.
- `HUMOR_GUIDE.md`: tono tragicómico permitido.
- `DATA_VISUAL_EVIDENCE_GUIDE.md`: reglas para visualización mínima.
- `CASE_RUBRIC.md`: rúbrica de calidad.
- `CODEX_WORKFLOW.md`: secuencia de skills.
- `CONTRIBUTING.md`: reglas de contribución.

## `.codex/`

- `operating_principles.md`: principios de operación para Codex.
- `task_protocol.md`: pasos para crear, mejorar o revisar casos.
- `skill_registry.md`: índice de skills locales.
- `skills/`: skills accionables para cada fase.

## `templates/`

Plantillas para caso, evidencia visual, datos sintéticos, HTML, revisión y nuevas skills.

## `prompts/`

Prompts listos para pedir a Codex casos desde tema, gráfica, problema de negocio, lote curricular o revisión.

## `examples/`

Casos publicados, HTMLs simples y especificaciones de datos/visualización.

## `tools/`

Herramientas Python mínimas para validar estructura, generar datos, generar HTML de gráfica y renderizar caso.

## `outputs/`

Destino local para archivos generados. No versionar salidas salvo `.gitkeep`.
