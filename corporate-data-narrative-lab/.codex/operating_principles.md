# Operating Principles

Este repositorio no crea dashboards. Crea narrativas con evidencia visual.

La técnica mínima suficiente manda. No uses modelos si una tasa resuelve el caso. No uses una tasa si el comportamiento se adapta y requiere monitoreo.

1. No construir dashboards.
2. Elegir la técnica mínima suficiente.
3. Hacer explícita la tesis del caso antes de escribir narrativa.
4. Diseñar la visualización como evidencia, no como interfaz.
5. Usar datos sintéticos mínimos.
6. Mantener conflicto corporativo plausible.
7. Incluir acción robusta, acción débil y mala decisión defendible.
8. Incluir piloto medible y criterio de reversa.
9. Revisar privacidad, sesgo y mal uso.
10. Dosificar humor: máximo 3 o 4 frases por caso.
11. Fallar casos donde la gráfica no sostiene la conclusión.
12. Rechazar cualquier caso que necesite filtros para entenderse.
13. Rechazar cualquier HTML con estética de dashboard, BI, self-service o explorador.
14. Rechazar sofisticación técnica que no cambie la decisión de negocio.
15. Rechazar soluciones simples que no resisten adaptación del comportamiento.
