# Project Brief

## Objetivo

Enseñar criterio de datos mediante comedia conversacional de oficina. El lector
primero reconoce una mala decisión corporativa y luego entiende qué debía mirar
en la gráfica.

## Audiencia

- estudiantes de ciencia de datos aplicada;
- analistas y equipos de negocio;
- docentes que necesitan casos listos para discusión;
- equipos que practican decisiones, no solo técnicas.

## Principios

1. La conversación manda; la gráfica prueba.
2. El humor nace de incentivos, burocracia y métricas.
3. La mala decisión es plausible bajo presión.
4. La persona que improvisó conserva dignidad.
5. Una sola gráfica basta.
6. Hay una sola pausa educativa, sin examen A/B/C.
7. Markdown es canónico y HTML es una vista simple.
8. Los datos son sintéticos.
9. La técnica mínima suficiente gana.

## No queremos

- reportes con chistes;
- narrador literario;
- jerga sin traducción;
- muletillas obligatorias;
- dashboards o navegación;
- objetos o meses actuando como personas;
- modelos usados por decoración;
- datos reales o sensibles.
