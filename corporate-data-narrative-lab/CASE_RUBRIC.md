# Case Rubric

## Fallo automatico

- no abre con dialogo;
- alguna intervencion no identifica personaje;
- un numero aparece sin decir que mide;
- usa `slide`, `deck`, `trae comparables`, `incrementalidad` o `escalaciones`
  dentro del dialogo;
- usa preguntas A/B/C;
- contiene mas de una grafica o ninguna;
- el remate principal se repite en otro caso;
- la tecnica aparece antes de que la escena necesite la grafica;
- culpa al trabajador bajo presion;
- el HTML agrega o quita contenido narrativo.

## Evaluacion

| Criterio | PASS |
| --- | --- |
| Naturalidad | Cada linea podria decirse en una oficina y responde a la anterior. |
| Contexto | Metricas, unidades y situacion se entienden sin prologo. |
| Humor | La mala logica escala y produce al menos un remate propio. |
| Critica | El sistema queda expuesto sin sermones. |
| Datos | La grafica cambia la lectura y sostiene la conclusion. |
| Aprendizaje | Una pausa natural y una explicacion breve bastan. |
| Ritmo | 450-700 palabras, 3-5 escenas y mayoria de dialogo. |
| Cierre | Decision concreta, remate seco y regla transferible. |

PASS requiere todos los criterios. NEEDS_REVISION significa que enseña pero
suena escrito, o que da risa pero la grafica no resuelve nada.
